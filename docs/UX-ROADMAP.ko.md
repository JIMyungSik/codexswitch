# codexswitch 사용성 개선 설계

## 목표

사용자가 내부의 인증 파일이나 로테이션 규칙을 몰라도 다음 세 가지를 즉시 이해해야 한다.

1. 지금 어떤 계정으로 실행되는가
2. 계정이 바뀌어도 무엇이 이어지고 무엇이 분리되는가
3. 문제가 생겼을 때 다음에 어떤 명령을 실행해야 하는가

## 핵심 사용자 흐름

### 첫 실행

`cxs`만 실행하면 긴 도움말 대신 설정 마법사를 제안한다.

1. 현재 Codex 로그인 가져오기
2. 두 번째 계정 추가
3. 공용 메모리 사용 여부 선택
4. 로테이션 임계치 확인
5. 계정 연결 상태 측정

완료 화면에서는 일상적으로 필요한 명령을 `cxs chat`, `cxs usage`, `cxs watch` 세 개만 보여준다.

### 평상시 실행

모든 실행 헤더는 같은 순서로 상태를 표시한다.

```text
account "work" · session continue · memory shared · reasoning concise
```

상태 용어는 `ready`, `paused`, `disabled`, `over threshold` 네 가지로 제한한다. 오류 메시지는 원인 다음 줄에 복구 명령을 제공한다.

### 계정 자동 전환

전환 전후를 하나의 이벤트로 표현한다.

```text
work paused for 1h 42m
↳ continuing the same session with personal (attempt 2)
```

세션 이력은 공유되지만 인증, 서버 측 ChatGPT Memory, isolated 메모리는 공유되지 않는다는 점을 최초 전환 때 한 번 안내한다.

## 메모리 모델

- `off`: 메모리를 프롬프트에 넣지 않는다. 기존 설치의 안전한 기본값이다.
- `shared`: 모든 계정이 `memory/shared.md`를 사용한다.
- `isolated`: 계정별 `memory/accounts/<name>.md`를 사용한다.

메모리는 `exec`, `run <계정> exec`, `chat`에만 명시적인 XML 경계로 주입한다. 인증 토큰이나 대화 원문을 자동 저장하지 않으며 사용자가 `memory add`로 넣은 내용만 저장한다. 파일은 32 KiB까지만 프롬프트에 포함해 컨텍스트가 끝없이 커지는 것을 막는다. 계정을 삭제해도 메모리 파일은 보존하며, 이름을 변경하면 isolated 메모리 파일도 함께 이동한다.

## 우선순위

### P0 — 현재 구현

- 한글 폭을 고려한 표 정렬
- 사용 가능 계정과 다음 계정 요약
- 채팅 턴·세션·추론 상태 표시
- 여러 줄 입력
- `off/shared/isolated` 메모리
- 프롬프트 보존과 전환 회귀 테스트
- 요청·계정 전환·결과·변경 파일을 연결한 구조화된 작업 이력

## 작업 이력 UX

`cxs log`는 인증과 로테이션 같은 운영 이벤트에 사용하고, `cxs history`는 사용자가 요청한 실제 작업을 보여준다. 두 개를 섞지 않아 목록을 읽는 목적이 분명하도록 한다.

```text
id          started           result  account flow       time  request
m5ab-2xk    2026-07-12 14:20  done    work → personal    42s   결제 모듈 테스트 수정
```

`cxs history show <ID>`는 요청 원문, 계정별 성공·한도·인증 실패, 실행 설정, 해당 실행 중 실제로 상태가 바뀐 파일, 작업 후 Git 상태를 구분해 표시한다. 실행이 끝날 때 상세 화면을 다시 찾을 수 있는 명령을 바로 출력한다. 이력 파일은 프롬프트를 포함하므로 경로와 보안 특성을 목록에 항상 표시한다. 요청은 최대 16 KiB까지만 저장한다.

### P1 — 다음 릴리스

- `cxs setup` 대화형 설정 마법사
- `cxs doctor`로 Codex 설치, 인증, 쓰기 권한, 세션 공유 상태 진단
- 오류마다 실행 가능한 복구 명령 한 개 표시
- 좁은 터미널에서 이메일·날짜 열을 자동 축약

### P2 — 이후 검토

- `watch`에서 계정 순서를 키보드로 변경
- 메모리 항목 검색·삭제·프로젝트별 범위
- JSON 출력 모드로 외부 UI 연동
- 선택형 로컬 웹 대시보드

GUI를 먼저 추가하기보다 설정 마법사와 진단 명령을 우선한다. 이 도구의 핵심 사용 장소가 터미널이므로 설치와 오류 복구가 쉬워지는 편이 실제 사용성 개선 효과가 더 크다.
